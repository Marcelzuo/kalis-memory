## Chrome 缓存攻坚（2026-06-12 08:39-08:55）

### 真凶
- Chrome tab 显示的 favicon **优先走 Google S2 CDN** (`gstatic.com/faviconV2`)，不是直接拉服务器
- Google 没爬过 / 缓存命中失败 → 显示 Google 默认 G 字母（不是旧 favicon，是占位符）
- 跟踪链路：`google.com/s2/favicons?domain=kalistorik.com` → 301 → `t0.gstatic.com/faviconV2` → 301 → `t1.gstatic.com/faviconV2` → 16x16 PNG 726B（Google 默认 G 字母）

### 验收三关（缺一不可）
1. **PIL 像素采样**：4 角 alpha=0 透明，中心 (0,0,0,255) 黑 — 验证密度
2. **ASCII 字符识别**：32x32 / 180 像素网格输出字符图，肉眼/脚本判 K 形状 — 验证形状
3. **vision 白底图判定**：透明 PNG 填白底后送 vision model，必须判 "K" — 验证语义
- **昨日漏洞**：只过第 1 关像素密度，没过第 2/3 关形状 — 教训入全局 MEMORY.md

### 解法
- HTML `<link rel="icon">` 加 `?v=20260612` → Chrome 看到新 URL 重新拉服务器 → 本地 SQLite 永久更新
- 24-72h Google 自然爬会更新 Google S2 缓存（新标签页/书签栏 G → K）
- 想加速 → Google Search Console 提交 URL
