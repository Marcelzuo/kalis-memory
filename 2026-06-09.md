## Contact 邮箱系统 Bug（6/9 16:53 老板报）

**症状**：
- Chrome：能发，但邮件**正文全空**
- Safari：**完全不能发**，无任何反应

**调查结论**（撒旦 16:53-17:05）：
- 后端 `/api/contact` 完全正常（curl 200，邮件真能到 hello@kalistorik.com）
- nginx 路由 / Python server (127.0.0.1:8081) / Zoho SMTP / CORS 头**全绿**
- **根因**：6/5 → 6/9 之间（v15.0/v15.0.1 某次改动）给 form 加了 mailto 兜底 + enctype="text/plain"，但兜底代码有 bug

**Bug 细节**：
1. line 968 form: `action="mailto:hello@kalistorik.com" method="post" enctype="text/plain"`
2. line 1999-2046 JS: fetch /api/contact 失败时，catch 块 `window.open('mailto:...?body=Name=xxx&Company=...&...')`
3. **Chrome 行为**：走 mailto 后，body 内 `&` 没编码成 `%26` → 邮件客户端把 `&Company=...` 当成新 header → 邮件主题能填，正文被吞
4. **Safari 行为**：`window.open('mailto:...')` 被 Safari 弹窗拦截器拒绝 → 用户看不到任何反馈 → "完全不能发"

**关键发现（v15.0.1 vs 6/5 备份对比）**：
- 6/5 v14.x 备份：form 无 action / 无 mailto 兜底 / JS 失败显示"Submission failed"
- 6/9 v15.0.1：form 有 mailto action / JS catch 走 mailto 兜底（**新引入**）
- 6/9 13:22 favicon 改动版（当前 dirty `f4f5ecc8`）= v15.0.1 + favicon 替换，**contact 代码无变化**

**6/9 13:22 改动验证**：

## Safari consent-checkbox 显示不一致（6/9 16:54 老板报）

**根因（已查清）**：
- line 211 CSS：`.consent-checkbox{display:flex;align-items:center;justify-content:center;flex-wrap:nowrap;white-space:nowrap;gap:0.4rem}`
- line 977-980 HTML：`<div class="consent-checkbox"><input checkbox>...<label><span>I agree to the <a>Privacy Policy</a></span></label></div>`
- **Safari 把 `<input>` 和 `<label>` 内的 `<a>` 当 2+ 个独立 flex item，justify-content:center 让它们分散对齐；Chrome 把 input+label 当 1 个组**
