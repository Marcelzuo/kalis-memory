
## 教训：撒旦越权做代码

- 图册和网站HTML不应该由撒旦单独完成
- 图片路径反复出错，因为撒旦不擅长代码细节
- 老板明确要求：所有任务三方参与，充分发挥路西法（认知）和玛门（执行）的能力
- 撒旦的职责：统筹、定方向、定规范；不写复杂代码
- 玛门的职责：代码执行
- 路西法的职责：设计审查、视觉把关

## 正确协作流程
1. 撒旦 → 整理需求、设计规范、产品数据
2. 玛门 → 写代码生成图册/网站
3. 路西法 → 审查设计，提修改意见
4. 撒旦 → 验收交付


## 诊断：玛门（Claude Code）输出卡住的根因

- 不是玛门的能力问题，是 cc-switch v3.15.0 中间代理
- cc-switch 有过异常退出记录，崩溃后API连接断开
- DeepSeek beta协议端点流式格式与Anthropic原生有差异，cc-switch需缓冲完整响应再转发
- maxTokens=384000 太大，大响应导致缓冲阻塞，表现为"卡住无输出"

## 修复
- deepseek-v4-pro: maxTokens 384000 → 64000
- deepseek-v4-flash: maxTokens 384000 → 64000
- 后续待办：检查cc-switch更新；考虑 Anthropic API 直连作为 fallback

## 协作模式确认
- 小代码任务 → `claude -p` 直接调用（够用）
- 大代码输出 → 谨慎使用，可能被缓冲卡
- 玛门首次调用成功生成了初版（结构完整），第二次大脚本被缓冲吞掉
