## 一、LinkedIn 安全边界 ⚠️ 最严

### 1.1 硬限制（违反 = 封号）

| 操作 | 免费账号上限 | 说明 |
|------|------------|------|
| **加好友** | ~100-150 个/周 | LinkedIn 不公布精确数字，实测安全线 20 个/天 |
| **Search 搜人** | 每月有 Commercial Use Limit（商业搜索额度） | 免费号 ~100 次商业搜索/月，用完降为有限结果 |
| **看 Profile** | 无公开上限但有隐形限制 | 短时间内大量看 Profile 触发"你是猎头/爬虫"标记 |
| **发 InMail/DM** | 免费号只能给已连接好友发 | 陌生人只能通过 Connection Request 附言（300 字符） |
| **创建公司页** | 需个人号 ≥10 好友 | 这是 LinkedIn 硬规定 |

### 1.2 安全操作守则

```
✅ 每天最多加 15-20 个好友（分散在全天，别一口气）
✅ 加好友必须带个性化备注（拒绝默认 "I'd like to add you"）
✅ 备注要真实、不重复、不模板化
✅ 搜索后至少等 3-5 秒再看下一个 Profile
✅ 新号前 2 周行为要像「真人」：看帖、点赞、评论，不要只加人

❌ 禁止批量操作（任何频率 >1 次/秒的动作）
❌ 禁止用同一个备注文案发给多人
❌ 禁止短时间内大量搜人（触发 Commercial Use Limit）
❌ 禁止新号第一天就大量加人（LinkedIn 对新号审查最严）
```

### 1.3 KALIS TORIK LinkedIn 安全分解

| 阶段 | 天数 | 每日操作 | 累计好友 |
|------|------|----------|----------|
| 热身期 | Day 1-3 | 完善 Profile + 看 5 个行业帖 + 点赞 3 个 + 加 5 个认识的人 | 5-15 |
| 温和期 | Day 4-7 | 每天加 10 个（带备注）+ 评论 2 条行业帖 | 35-55 |
| 正常期 | Day 8+ | 每天加 10-15 个 + 发 1 条帖 | 100+ |

## 二、Instagram 安全边界

### 2.1 硬限制

| 操作 | 上限 | 说明 |
|------|------|------|
| **关注** | ~200 个/天 | 新号更低，建议 50-80 个/天 |
| **取关** | ~200 个/天 | 关注后短期内取关 = 明显机器人行为 |
| **点赞** | ~1000 个/天 | 但新号 200-300/天更安全 |
| **评论** | ~200 条/天 | 每条评论必须不同、有意义 |
| **帖子** | 无硬上限 | 但新号每天 1-2 帖最安全 |
| **Story** | 无硬上限 | 正常使用即可 |
| **DM** | ~50-100 条/天 | 新号必须更低 |

### 2.2 安全守则

```
✅ 新号前 7 天行为要像「真人」：刷 Explore、看帖、偶尔点赞
✅ 关注/取关比例 > 2:1（关注多，取关少）
✅ 每小时操作间隔至少 5 秒
✅ 评论要有意义且不重复（IG 检测重复文本）
✅ IP 地址稳定（不要频繁切换 VPN）

❌ 禁止 1 小时内关注 >30 人
❌ 禁止关注后立刻取关
❌ 禁止重复评论
❌ 禁止 24 小时连续操作（真人会睡觉）
```

### 2.3 KALIS TORIK IG 安全节奏

| 阶段 | 天数 | 每日操作 |
|------|------|----------|
| 热身期 | Day 1-7 | 完善 Profile + 发 3 帖 + 每天手动互动 15 分钟（看帖、点赞、收藏） |
| 温和期 | Day 8-14 | 每天关注 20-30 人 + 评论 5 条行业帖 + 正常发帖 |
| 正常期 | Day 15+ | 每天关注 50-80 人 + 评论 10 条 + 正常发帖节奏 |

## 三、Facebook 安全边界

### 3.1 硬限制

| 操作 | 上限 |
|------|------|
| **加好友** | ~1000 个好友总数（个人号上限 5000） |
| **每日加好友** | 建议 20-30 个/天，超过可能触发验证 |
| **加群组** | ~10-20 个/天 |
| **发帖** | 新页面每天 1-2 帖 |
| **群组发帖** | 同一条消息不发多于 3 个群组 |

### 3.2 安全守则

```
✅ 页面创建后前 3 天只完善信息、不大量发帖
✅ 加群组后先观察群规、互动数天再发自己的内容
✅ 在他人群组：先评论助人，再自然提及品牌
✅ IP 和设备保持稳定

❌ 禁止在群组直接甩链接（视为 spam，秒踢）
❌ 禁止同一条内容发多个群组（FB 检测跨群重复）
❌ 禁止买粉买赞（FB 算法精准识别并降权）
```

## 四、opencli 自动化安全适配

### 4.1 LinkedIn

```bash
# 搜人 — 安全（每月有 CUL 额度）
opencli linkedin people-search "furniture buyer europe"

# 读 Profile — 安全（别一口气读太多）
opencli linkedin profile-read https://linkedin.com/in/xxx

# 加好友 — ⚠️ 这是 UI 模式，会真的在浏览器里点击
# 每次调用 = 1 次真实操作，遵守 20/天 限制
opencli linkedin connect https://linkedin.com/in/xxx "Hi, also in furniture..."

# 发私信 — ⚠️ 仅发给已连接好友
opencli linkedin safe-send <thread_url> "Message text"
```

### 4.2 Instagram

```bash
# 发帖 — 安全
opencli instagram post "Caption" --media image.jpg

# 评论 — ⚠️ 遵守每日上限
opencli instagram comment <post_url> "Meaningful comment"

# 关注 — ⚠️ 遵守每日上限
opencli instagram follow <username>

# 搜人 — 安全
opencli instagram search "furniture sourcing"
```

### 4.3 Facebook

```bash
# 搜人/页 — 安全
opencli facebook search "furniture buyers europe" --type pages

# 加好友 — ⚠️ 遵守每日上限
opencli facebook add-friend <profile_url>

# 加群组 — ⚠️ 遵守每日上限
opencli facebook join-group <group_url>
```

## 五、操作前必查清单

执行任何自动化操作前，逐项检查：

```
□ 今天已操作次数 < 每日上限
□ 当前操作频率 ≥ 5 秒/次
□ 内容不重复（备注/评论每次不同）
□ 账号已完成热身期（新号等够天数）
□ IP 地址稳定、无频繁切换
□ 被操作对象是真人活跃账号（不加僵尸号——平台会关联惩罚）
```
