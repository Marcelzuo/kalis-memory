## 2. 阶段定义与流转规则

| 阶段 | 定义 | 准入条件 | 下一步动作 | 停留上限 |
|---|---|---|---|---|
| 新线索 | 初步接触，未沟通 | 来源渠道已确认 | 撒旦 24h 内发出第一条联系消息 | 3 天 |
| 已联系 | 已发出首次触达，未回复或初对话 | 至少有 1 条触达记录 | 等回复；3 天无回应触发二次触达 | 7 天 |
| 报价中 | 客户表达需求/兴趣，已发报价或方案 | 客户明确需求点 | 撒旦 3 天内跟进一次，确认收件+态度 | 14 天 |
| 谈判 | 价格/条款/交期来回沟通 | 双方均进入实质讨论 | 老板介入大单谈判（>€5000） | 21 天 |
| 成交 | 已签合同/付款/交付 | 凭证落地 | 移交交付团队，30 天后回访 | 30 天 |
| 丢失 | 客户明确拒绝/失联超 30 天 | 明确信号 | 写入丢失原因（必填），3 个月内不再主动联系 | 永久 |

阶段流转由路西法根据撒旦反馈写入。触发词："客户 X 进入报价阶段"、"A 客户成交"、"B 客户已丢失"。

## 3. 每周必做 — 跟进超时巡检

**执行人**：路西法（每周日晚 21:00 自动巡检，可手动触发）
**输出**：简报一条，推送到老板；含超时客户清单

巡检逻辑：
1. 扫描追踪表，计算 `今天 - 最近跟进日期`
2. 凡 > 7 天未跟进 → 高亮标红 + 列出该客户当前阶段
3. 处于"新线索/已联系"超 7 天 → **升级撒旦立即行动**
4. 处于"报价中/谈判"超 7 天 → **提醒老板介入决策**
5. 处于"成交"超 30 天未回访 → 提醒撒旦安排回访

格式：
```
[CRM 周报 YYYY-MM-DD]
⚠️  超时跟进 X 条：
  - 客户A（报价中, 10天未动）→ 提醒撒旦
  - 客户B（谈判, 14天未动）→ 请老板定夺
✅  本周推进：M 条进入下一阶段，N 条成交
```

## 5. 使用约定

- **谁填**：撒旦每天动作（联系/报价/成交）→ 反馈给路西法 → 路西法写入此表
- **脱敏红线**：禁止写真实姓名/电话/邮箱/地址；违例立即整改
- **来源缩写**：IG=Instagram / LI=LinkedIn / FB=Facebook / WA=WhatsApp / WEB=网站表单
- **Ledger 同步**：每次表更新必须同时记一条 ledger（路径：`/Users/zuo/Marcelzuo/kalis-memory/docs/ledger/YYYY-MM-DD.md`）
- **历史归档**：表行 >200 条时，把 >3 个月的记录移到 `archive/` 子目录，主表只保留活跃线索
