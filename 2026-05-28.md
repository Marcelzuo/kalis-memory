### 玛门 API Error 400 修复
- **根因**：DeepSeek V4 Pro 默认 thinking 模式，CC-Switch 不透传 thinking 块 → 多轮对话触发 400
- **修复路径**：
  1. settings.json 加 `alwaysThinkingEnabled: false` → CC-Switch 覆写，失败
  2. settings.local.json 加 → 部分有效但不稳定
  3. CC-Switch 配置 Anthropic 端点 → 仍有干扰
  4. **最终方案**：DeepSeek 直连 `api.deepseek.com/anthropic`，完全绕过 CC-Switch
- 切换脚本：`workspace/switch_mammon.sh deepseek|minimax|status`
- 直连配置：`~/.claude/settings.deepseek.json`

### 根本原因
- Claude Code v2.1.150 → v2.1.153 今天上午 11:12 更新
- 新版主动启用 thinking，暴露了 CC-Switch 的透传 bug
- DeepSeek V4 从发布起就默认 thinking，但旧版 Claude Code 没主动用

### 关键教训
- CC-Switch 每次保存都会覆写 settings.json
- `alwaysThinkingEnabled: false` 只管 Claude Code 客户端，管不到 DeepSeek 服务端 thinking
- DeepSeek Anthropic 端点 + CC-Switch 的组合仍会丢失 thinking 块
- 唯一的稳定方案：直连，不走 CC-Switch
