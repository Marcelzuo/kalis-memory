### 模块顺序的铁律

```
老板决策（第 1 章）→ 平台规则（第 2 章）→ 内容打法（第 3-4 章）
→ 执行落地（第 5-6-7 章）→ 安全底线（第 8 章）→ 工具与数据（第 9-10 章）
```

理由：老板先选方向，再往下看细节。不要把安全放在最后才想起来。

## 二、方案对比框架（A/B/C/D 四象限）

### 对比表格式（固定列）

| 方案 | 干法 | 每周时间 | 12个月核心指标 | 适合谁 |
|------|------|----------|----------------|--------|

- 必须列出 **老板时间投入**（老板最关心这个）
- 必须列出 **核心产出预估**（粉丝 / 询盘 / 其他）
- 预估必须标注假设前提（如「纯有机增长」「零广告」）
- 不编造数据，不确定的标 `~`

### 四象限设计逻辑（路西法原创）

```
              ↑ 重内容（深度）
              │
         A    │    D
 轻互动 ←─────┼─────→ 重互动
         C    │    B
              │
              ↓ 重视频（广度）
```

- **A**：深度内容 + 轻互动（适合不愿出镜）
- **B**：短视频轰炸 + 轻互动（适合想快速破圈）
- **C**：深度内容 + 重互动（适合急要询盘）
- **D**：混合（折中，都要）

### 决策清单（4 个必问问题）

1. 我愿意每周出现在镜头前吗？
2. 我能接受前 3 个月数据惨淡吗？
3. 我要品牌曝光还是实际询盘？
4. 我手上有多少真实客户素材可用？

每个问题 → 直接指向方案。不让老板猜。

## 三、平台硬规格调研规范

### 数据来源铁律
- **100% 来自官方帮助中心 / 开发者文档**，不推测
- 每条规格标注来源 URL
- 区分「平台上限」和「推荐值」

### 必查清单（每个平台）

| 内容类型 | 必查项 |
|----------|--------|
| **图文帖** | 最大分辨率、宽高比范围、多图上限、文字上限、标签上限 |
| **视频/短视频** | 宽高比、最低帧率、最低分辨率、时长范围、文件格式、封面规格 |
| **快拍/Story** | 宽高比、分辨率、时长、过期规则 |
| **主页/Page** | Bio 字数、链接数、CTA 按钮类型 |
| **直播** | 时长上限、分辨率、是否可回放 |
| **广告** | 图片/视频规格、文字占比限制、受众定向选项 |

### IG 示例

```
| 参数 | 规格 | 来源 |
|------|------|------|
| 最大宽度 | 1080px | help.instagram.com/1631821640426723 |
| 宽高比 | 1.91:1 ~ 3:4 | 同上 |
| 多图轮播 | 最多 20 张 | Instagram 公开文档 |
```

## 四、内容哲学框架

### 4.1 为什么传统打法必死（米迦勒原创）

**模板**：
> 传统 [行业] 的 [平台]：[常见做法 1]、[常见做法 2]、[常见做法 3]。
> 客户刷到的反应只有一个——"[客户内心 OS]"。

然后引出你的差异化打法。

### 4.2 心理触发器 9 条（撒旦原创）

| # | 触发器 | 可用于的 hook |
|---|--------|---------------|
| 1 | 好奇心缺口 | "他没猜到的事" |
| 2 | 反差感 | "差点被退货 → 客户加单" |
| 3 | 情绪共鸣 | "如果你也经历过 X，看这条" |
| 4 | 故事化 | 完整 narrative arc |
| 5 | 数字震撼 | 具体数字 vs 模糊承诺 |
| 6 | 真实感 | 手持拍摄、不修图 |
| 7 | "懂你"角度 | "If you've ever X..." |
| 8 | 独家视角 | "我们故意 X——然后发现 Y" |
| 9 | 未完成感 | "3 个月后再发对比——先关注" |

### 4.3 五大爆火内容框架（米迦勒原创）

| 框架 | 心理机制 | Hook 公式 |
|------|----------|-----------|
| 🔥 真相炸弹 | 认知失调 | "YOUR [角色] IS LYING TO YOU" |
| 🔥 禁忌画面 | 窥探欲 | "Most [同行] won't show you this" |
| 🔥 行业黑幕 | 打破潜规则 | "Why your [成本] costs more than it should" |
| 🔥 失败自白 | 脆弱性=可信度 | "**我们**搞砸过" |
| 🔥 FOMO 轰炸 | 竞争焦虑 | "A [同行] just switched. Here's what changed." |

### 4.4 内容 → 商业目标映射（路西法原创）

| 商业目标 | 衡量指标 | 平台内容触点 | 引流终点 |
|----------|----------|-------------|----------|
| 品牌曝光 | 粉丝/触达 | [帖子类型] | Bio Link / Profile |
| 询盘转化 | DM/邮件数 | 强 CTA / Lead Magnet | DM / Email / WA |
| 信任建立 | Saves/Shares | 客户故事 / insight | Saves → Profile → Link |
| 行业权威 | Saves 增长率 | insight / 时事 | Newsletter |

## 五、品牌独有资产提取

### USP 提取方法
- 从真实业务中提炼，不编造
- 每条 USP 必须可验证（有案例 / 有数据 / 有客户名）
- 区分「行业通用」和「KALIS 独占」

### 痛点清单提取方法
- 列出目标客户真实的 B2B 痛点
- 每个痛点 → 必须有 KALIS 的对应解法
- 用老板原话或客户原话，不润色成营销语言

## 六、执行日历规范

### 12 周分阶段（漏斗模型）

```
W1-W4   热身期（SHOCK）   低调、不主动 DM、建基础内容
W5-W8   信任期（TRUST）   开始轻互动、客户故事、insight
W9-W12  转化期（CONVERT） 强 CTA、主动拓展、案例加码
```

### 热身期强制规则
- 前三周只看不发 DM
- W4 开始 ≤ 3 条主动 DM/天
- 不关注后立即取关
- 不评论 "DM me" "Check my page"

### 每周节奏模板

```
周五 路西法 → 提交下周选题 + 脚本
周六 撒旦初审 → 修改 → 玛门出图/视频
周日 撒旦 + 米迦勒双签 → 老板终审
周一/三/五 米迦勒按日历发帖
每日 米迦勒互动候选 → 老板批 → 执行
周日 米迦勒出 KPI 周报
```

## 七、视觉与表达规则

### 品牌 VI 铁律
- 色彩：黑 #000000 / 白 #FFFFFF / 金 #C9A96E / 灰 #888888
- 字体：Montserrat（Google Fonts），不换不压不改
- Logo 顺序：金线在最上 → KALIS → TORIK

### 文案节奏（3 段式）
```
Hook（1 行） + 痛点（2 段） + USP（1 段） + CTA（1 行）
```

### Hashtag 策略
- 品牌专属 3 个
- 泛行业 5 个
- 细分产品 5 个
- 痛点类 3 个
- 地域/客户 3-5 个
- 总量控制在平台的 60-75%（不顶满）

## 八、安全合规（防封号铁律）

### 必含内容
1. 平台社区规范引用（官方 URL）
2. 新号限制（DM 数、关注数、评论数/天）
3. 禁止行为清单（群发、冷 DM、第三方工具、敏感词）
4. DM 话术模板（必须提及对方具体内容）
5. 热身期逐步放开时间表

### 安全铁律
- 安全规则优先级 > 增长目标
- 封号 = 全盘皆输
- 所有互动操作设日上限

### 周级追踪表

| 指标 | W1 | W2 | W4 | W8 | W12 |
|------|----|----|----|----|-----|
| 粉丝 | >X | >Y | ... | ... | ... |
| 互动率 | ... | | | | |
| 询盘 | ... | | | | |

### 关键决策节点
- W4 末：复盘 → 调整方向
- W8 末：复盘 → 深化或转向
- W12 末：总复盘 → 下一轮策略

### To B 核心原则
> 询盘质量 > 粉丝数量。一个成交客户的价值远超粉丝数。

## 十、多人协作规范

### 三区分工（v3.3 铁律）

| 区 | 谁 | 做什么 | 禁止 |
|----|-----|--------|------|

## 十一、老板反馈接收规范

### 老板反馈的处理流程
1. 老板反馈 → 定位是哪个模块的问题
2. 如果是方向性推翻（如「工厂 40%+产品 35%+知识 25%=死路」）→ 推倒重做该模块
3. 如果是细节修正 → 追加版本履历
4. 反馈闭环后报告老板

### IG 案例
- 老板 v1 反馈「死路」→ 地狱三人组 v2 完全重做内容哲学
- 老板拍板「零广告」→ v3.2 全删广告模块
- 撒旦 review 3 处 → v3.2 逐条修正
